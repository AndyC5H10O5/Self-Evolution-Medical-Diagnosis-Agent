# 自进化工具化 — 实验验证清单（半自动）

用于验证：**仅主流程模型**能否在合适时机通过 `tool_calls` 调用 `skill_evolve`，且不误触发。

## 实验前准备

1. 配置好 `.env`，启动：`python src/agent_core/main.py`（或 `scripts/run_main.ps1`）。
2. 建议先用 **有 `evolvable_fields` 的症状**（如头痛 `headache`）做完整一轮；必要时对 `skills/headache/SKILL.md` 做备份或 `git stash`。
3. 终端观察日志行：`[tool_call] skill_evolve`、`[tool_call] save_document`。
4. 建议用下表记录每场景一行（可复制到 Excel / Notion）。

### 记录表头（可复制）

```text
场景ID | 是否出现skill_evolve | 是否符合预期(TP/TN/FP/FN) | 工具返回摘要 | SKILL.md是否变更
```

---

## A 组 — 应触发 `skill_evolve`（正例）

| ID | 说明 | 预期 |
|----|------|------|
| A1 | 模型在问「头痛性质」时，用户给出**原 SKILL 括号列表里没有**的短词（2–12 字） | 出现 `skill_evolve`，且 `field_label` 为性质相关字段；对应行写入新词 |
| A2 | 模型在问「伴随症状」时，给新短词（列表无） | 同上，`field_label` 为伴随症状字段 |
| A3 | 模型在问「头痛部位」时，给新部位词（列表无） | 同上，部位字段 |

---

## B 组 — 不应触发（负例）

| ID | 说明 | 预期 |
|----|------|------|
| B1 | 纯闲聊、情绪、与选项无关 | **无** `skill_evolve` |
| B2 | 回答实质等价于已有选项（列表已覆盖） | **无**调用为佳；若调用应为 `No update` |
| B3 | 超长短语或含 `/`、`(`、`)`、`（`、`）` | **无**调用为佳；若调用工具应返回 `Error:` |
| B4 | 正在头痛问诊，用户突然转向无关器官主诉且非可沉淀「选项」 | **无**对当前 skill 的追加 |

## C 组 — 工具层稳健性（可选，不依赖模型）

cd e:\agent\Consult_Medical_Agent

- 第二次执行同一 `new_option` 应出现 `No update`。
- 将 `field_label` 改为非允许字段应出现 `Error` 类提示。

## 半自动「脚本」— 手工逐条粘贴（头痛 skill）

以下按顺序复制到 REPL **每一行或每一段**发送：

### 脚本 1：正例为主（性质 / 伴随 / 部位）

```text
我头痛三天了，时轻时重。
```

（等待模型提问后，若问性质或类似，下一条：）

```text
像戴了紧箍咒一样，发紧发胀，不是一跳一跳那种。
```

（等待下一问；若问部位或伴随，按需选用下面之一：）

```text
主要是后脑勺连着脖子那一块。
```

```text
闻不了油烟，闻到就想吐。
```

（可选：结束语）

```text
先问到这，请把目前信息整理成简短预问诊记录并保存到本地。
```

**记录：** 是否在性质/伴随/部位相关回答之后出现 `skill_evolve`；参数是否合理。

---

### 脚本 2：负例 — 闲聊与跑题

先激活头痛语境：

```text
我偏头痛反复发作。
```

（等模型回复后：）

```text
我好害怕，会不会是绝症啊，医生你在吗？
```

（再等回复：）

```text
对了，我肚子也绞痛，跟头痛没关系，你先别管肚子。
```

**记录：** 上述几句之后是否**错误**出现 `skill_evolve`（理想为全程无）。

---

### 脚本 3：边界 — 过长或非法字符（模型可能误调工具）

在模型正在问某一组「选项类」问题时发送：

```text
我这个疼法是那种说不清楚的很复杂的一种混合疼还有放射还有麻木持续一整天那种。
```

或（含斜杠，工具应拒绝）：

```text
性质是刺痛/灼烧混合。
```

**记录：** 是否调用工具；若调用，返回是否为 `Error:` 或模型是否克制不调用。

---

### 脚本 4：短会话 — 仅测保存

```text
我偶尔头疼，不严重，没有别的症状。请直接给我一份很短的预问诊小结并保存。
```

**记录：** 是否仅有 `save_document`（或合理无进化）。WW

## 汇总指标（科研向）

- **召回（正例）**：A 组中「应触发且写对字段」的比例。
- **精确（触发）**：所有 `skill_evolve` 中，人工判定「确实该进化」的比例。
- **安全**：B/C 组中导致错误写盘或错误字段的次数（理想为 0）。
