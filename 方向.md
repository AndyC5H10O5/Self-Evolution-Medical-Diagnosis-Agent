**会议主题**：医疗问诊Agent自进化系统毕设指导
**发言人**：王汉唐22331090、陈文清
**会议摘要**：本次会议针对王汉唐的医疗问诊 Agent 系统毕设进行了指导，重点解决了系统缺乏评价体系的问题，明确了以 Benchmark 验证自进化效果为核心的研究方向，并探讨了 Agent 框架的未来价值。
---
### 一、系统现状与核心问题诊断
针对王汉唐开发的医疗问诊 Agent 系统，会议首先梳理了当前的技术实现与存在的核心缺陷：
#### 1. 系统架构与功能实现
- **核心控制逻辑**：系统参考了 OpenClaw 的源码，实现了 Agent Loop 作为核心控制循环，并在此基础上开发了 Skills 相关的进化能力。
- **混合检索机制**：设计了“关键词命中 + 语义检索”的混合检索方法，用于从 Skills 中读取规则并注入 Prompt。
- **动态进化能力**：实现了细粒度的自进化，支持在问诊过程中根据用户输入动态拓展高价值问题及引导选项（如新增“触痛”选项）。
#### 2. 当前面临的核心瓶颈
- **缺乏评价体系**：系统目前仅实现了功能，但缺乏科学的 Benchmark 来衡量“自进化”是否真正提升了问诊效果（如覆盖率、准确性）。
- **进化触发逻辑模糊**：目前的进化触发条件较为宽松，仅基于用户输入是否在现有选项之外进行判断，缺乏基于“回答正确/错误”的反向修正机制。
- **工程与科研脱节**：学生目前侧重于工程实现，尚未将系统置于学术研究框架下进行验证，导致论文缺乏数据支撑。
### 二、毕设优化方向与实施路径
陈文清老师针对上述问题，提出了具体的改进方案，旨在将项目从单纯的工程实现转化为符合学术规范的毕业设计：
#### 1. 建立科学的评价体系
- **引入 Benchmark 数据集**：建议停止从零构建评价指标，转而寻找现有的医疗问诊 Benchmark 或构建标准数据集（Ground Truth）。
- **数据划分与验证**：建议将数据集划分为训练集（70%）和测试集（30%），利用训练集进行 Skill 的自进化训练，并在测试集上验证进化后的表现。
- **明确进化价值**：必须证明动态添加的选项（如“触痛”）能够辅助最终的问诊效果，而非单纯的功能堆砌。
#### 2. 优化自进化触发逻辑
- **基于反馈修正**：改变当前宽松的触发机制，设计基于“回答正确/错误”的比对逻辑。通过分析错误案例（如病因关联错误、用药建议错误），将缺失的经验补充到 Skills 中。
- **弱化人工干预**：建议减少对人工调优的依赖，利用模型自身能力通过多步思考来预测意图，实现自动化的经验沉淀。
### 三、论文撰写与工程边界界定
针对学生关于“完成度”和“创新性”的担忧，会议明确了论文的撰写重点与工程实现的取舍：
#### 1. 论文结构规划
- **双核心章节**：论文应包含“系统架构设计”（讲述前后端框架、Agent 设计）和“自进化算法实现”两个核心章节，两者结合即可构成完整的毕设内容。
- **创新性要求**：明确本科毕设不强制要求理论创新，重点在于完整复现一个流程（将自进化应用于 Benchmark 并验证效果），并阐述其应用价值。
#### 2. 技术选型与取舍
- **核心聚焦**：建议将精力集中在“Skill 自进化”这一亮点上，其他模块（如 RAG、MCP 记忆）属于锦上添花，有时间可做，无时间可略。
- **工具化封装**：若涉及外部知识检索，可考虑封装为 MCP 协议的工具，但这不是毕设的强制要求。
### 四、行业洞察与技术探讨
会议最后针对学生提出的关于 Claude Code 源码及 Agent 框架价值的疑问进行了延伸讨论：
#### 1. Claude Code 源码价值分析
- **设计理念重于代码**：指出 Claude Code 的强大主要源于其基座模型能力和海量 Agent 训练，源码本身与 OpenClaw 等开源框架类似，重点应关注其背后的 Memory 管理、Tool 设计及 Sub-agent 规划理念。
- **个性化助理方向**：Claude Code 的目标是打造个性化助理，这与 OpenClaw 有重叠，可关注其实现路径。
#### 2. Agent 框架的未来趋势
- **框架价值贬值**：随着基座模型能力的增强，Agent 框架本身的价值会逐渐贬值，复杂的 Prompt 工程可能被简化为轻量级提示。
- **不可替代性**：Memory（记忆管理）和 Tools（未知工具管理）是 Agent 框架长期存在的核心价值，因为基座模型无法将动态更新的外部知识固化到参数中。
